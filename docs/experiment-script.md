# Experiment Script

## 1. Opening Script

Thank you for helping us test the Smart Campus Event Hub prototype. This is not a test of your abilities. We are testing whether our product idea and interface are easy to understand and use. Please perform each task as naturally as possible and think aloud while using the prototype. Your feedback will help us improve the system before developing the final prototype.



## 2. Test Scenario

Imagine you are a university student who wants to discover upcoming campus events, register for an event, and keep track of your participation. Use the Smart Campus Event Hub prototype to complete the following tasks.



## 3. Tasks for Tester

| Task No. | Task Instruction | Related Requirement | Expected Result |
|---|---|---|---|
| T1 | Open the homepage and explain what the Smart Campus Event Hub is used for. | FR-01, FR-02 | User understands the purpose of the system and identifies the main navigation options. |
| T2 | Browse the list of available campus events and use the search or category filter to find a specific event. | FR-05, FR-06 | User successfully finds a relevant event using the search and filter functions. |
| T3 | Open an event and review its details, including the date, time, venue, organizer, and registration status. | FR-07, FR-08 | User understands the event information and current registration status. |
| T4 | Register for an event by completing the registration form. | FR-03, FR-10, FR-11 | User successfully submits the registration form and receives a confirmation message. |
| T5 | View the dashboard and explain the event summary information, or (if testing as an organizer) open the Admin page and update an event. | FR-09, FR-12 | User understands the dashboard summary or successfully performs an event management action. |



## 4. Observation Notes

Record any observations during the usability test, including:

- Where the tester hesitates or appears confused.
- Whether the tester clicks the wrong button or follows an unexpected path.
- Questions asked by the tester during the tasks.
- Any navigation problems or usability issues.
- Suggestions or comments provided by the tester.


## 5. Closing Questions

1. What was easy to understand when using the Smart Campus Event Hub?
2. Was there anything confusing or difficult to use?
3. Which feature did you find most useful?
4. Was it easy to find and register for an event?
5. Do you have any suggestions for improving the prototype before the final version is developed?
