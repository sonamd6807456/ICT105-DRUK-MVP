# Lab 08 - MVP Decision

## 1. Decision

- [x] Continue with minor revisions

---

## 2. Evidence Supporting the Decision

The Smart Campus Event Hub MVP was validated with **10 test users**, including students, international students, a student club organizer, and a campus administrator.

The usability testing results showed:

- **Task Success Rate:** 85%
- **Average Ease of Use Score:** 4.5/5
- **Average Usefulness Score:** 4.8/5

Most participants were able to complete their assigned tasks, including browsing campus events, searching events, viewing event details, registering for events, accessing the student dashboard, and testing event management features.

The validation results showed that the Smart Campus Event Hub successfully addresses the problem of scattered campus event information by providing a centralized platform for students and organizers to discover and manage university activities.

Users gave positive feedback on the Event Listing, Event Details, Event Registration, Dashboard, and Event Management features. These features helped users easily access event information and complete important tasks.

However, some users experienced difficulties with the Search and Filter feature, registration form clarity, dashboard organization, and event category selection. These issues affected the usability experience and require improvements before the next development stage.

Based on the validation results, the team decided to continue development with minor revisions to improve usability, navigation, and overall user experience.

## 3. Requirements to Keep

| Requirement ID | Reason |
|---|---|
| FR-01 | The Event Listing feature successfully helped students discover upcoming campus events. Users found the homepage and event list easy to navigate. |
| FR-03 | Search and filtering are important for event discovery. Although improvements are needed, users found the feature useful for locating events. |
| FR-04 | The Event Details page provided complete event information including descriptions, locations, and schedules. |
| FR-05 | The Event Registration feature successfully supported student participation in campus events. |
| FR-08 | The Student Dashboard helped users view upcoming events and important event-related information. |
| FR-09 | Event Management supported organizers and administrators in creating and managing campus events. |
| FR-10 | Updating event information was completed successfully and supported event administration tasks. |

---

## 4. Requirements to Improve

| Requirement ID | Problem Found | Improvement Needed |
|---|---|---|
| FR-03 | Some users found search categories and filters difficult to understand. | Improve filter labels and add more filtering options such as date, organizer, and event category. |
| FR-05 | Some users suggested clearer registration form instructions and field labels. | Improve registration form layout and make the registration process easier to understand. |
| FR-08 | Dashboard information was useful but could be organized more clearly. | Improve dashboard structure and information arrangement. |
| FR-09 | Organizers experienced difficulty selecting event categories during event creation. | Simplify event submission forms and improve category selection. |

---

## 5. Prototype Changes Before Next Lab

The team will implement the following improvements:

- Improve search and filtering functionality.
- Add clearer event category labels.
- Add additional filtering options for better event discovery.
- Improve event registration form instructions.
- Improve student dashboard layout and information organization.
- Simplify organizer event creation and management workflow.
- Conduct another usability testing round after implementing improvements.
