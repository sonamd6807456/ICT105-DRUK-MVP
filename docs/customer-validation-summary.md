## 4. Validation Method

The prototype was evaluated through task-based usability testing. Each participant was asked to complete a series of common tasks while observers recorded task completion, time taken, user comments, and any difficulties encountered.

- **Testing method:** Task-based usability testing
- **Date/time:** July 2026
- **Location or online platform:** Rangsit University Computer Laboratory
- **Number of testers:** 10
- **Data collected:**
  - Task completion status
  - Task completion time
  - User feedback
  - Satisfaction score
  - Interest level
  - Navigation and usability observations

---

## 5. Summary of Results

| Metric | Result | Interpretation |
|---------|--------|----------------|
| Total test users | 10 | Larger sample provides stronger validation of the MVP |
| Task success rate | 100% (10/10) | Every participant successfully completed the assigned tasks |
| Average feedback score | 4.8 / 5 | Users were highly satisfied with the prototype |
| Average interest level | 4.9 / 5 | Students expressed strong interest in using the platform |
| Most common confusion point | Event registration button placement | Minor interface improvements can enhance usability |

---

## 6. Key User Comments

Most participants found the website easy to navigate and appreciated its clean layout. The event listings were well organized, making it simple to browse available activities. Users also found the signup and login process straightforward and liked the responsive design across different screen sizes.

Several participants suggested making the **Register** button more noticeable and adding a search bar and event category filters to improve navigation. Some users also recommended displaying a clearer confirmation message after successfully registering for an event or submitting a review.

---

## 7. Affected Requirements

| Requirement ID | Evidence Found | Required Prototype Improvement |
|----------------|----------------|--------------------------------|
| FR-03 | Users requested quicker ways to locate events | Add search and filtering options |
| FR-04 | Some users overlooked the Register button | Improve button size, color, and placement |
| FR-05 | Users wanted additional profile customization | Expand profile management features |
| FR-06 | Confirmation after review submission could be clearer | Add success notifications and confirmation messages |

---

## 8. Conclusion

The customer validation results demonstrate that the **Smart Campus Event Hub MVP is successfully validated**. All **10 participants** completed the required tasks without major issues, resulting in a **100% task success rate**. The prototype received an **average feedback score of 4.8/5** and an **average interest level of 4.9/5**, indicating that it effectively meets the needs of students seeking campus event information and registration. User feedback highlighted only minor usability improvements, such as enhanced event search, more prominent registration buttons, and clearer confirmation messages. These findings validate the current MVP while providing clear direction for future enhancements.
