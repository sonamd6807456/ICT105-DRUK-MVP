# Analytics Insights

## 1. Analytics Question

**What did users do, say, and prove when testing the Smart Campus Event Hub MVP direction?**

The validation aimed to evaluate whether students, organizers, and campus administrators could successfully use the Smart Campus Event Hub prototype to discover campus events, search and filter events, view event details, register for events, save favorite events, access dashboards, and manage event information with minimal assistance.

The purpose of this validation was to determine whether the MVP successfully solves the problem of scattered campus event information across multiple platforms such as social media, messaging applications, and campus announcements.

The validation results provide evidence of user behavior, usability challenges, and opportunities for improving the next prototype version.

# 4. Interpretation

The validation results indicate that the Smart Campus Event Hub MVP successfully addresses the problem of scattered campus event information. Most participants were able to browse events, search activities, view event details, register for events, save favorite events, and access dashboards.

The prototype achieved an **86% task success rate**, showing that the overall workflow is understandable and usable. However, the remaining unsuccessful tasks highlight areas where users experienced difficulties, especially with searching, filtering, and understanding some management features.

Users rated the platform positively with an average usefulness score of **4.7/5** and an average interest level of **4.7/5**. These results demonstrate that users recognize the value of having a centralized platform for discovering and managing campus events.

The main usability problems identified were related to the Search and Filter feature, event category selection, registration form clarity, and dashboard organization. These issues do not affect the core MVP concept but require interface improvements before further development.

Overall, the analytics provide evidence that the Smart Campus Event Hub MVP direction is validated.

# 6. Next Prototype Improvement

Based on the validation results, the next prototype version should focus on usability improvements.

Priority improvements include:

- Improve Search and Filter functionality.
- Add clearer event categories and filtering options.
- Improve event registration form instructions.
- Enhance Student Dashboard organization.
- Simplify Admin Dashboard management workflow.
- Improve event creation and editing features.
- Maintain the current successful event browsing experience.
- Conduct another usability testing round after improvements.
- Update prototype documentation and GitHub repository.

# MVP Decision

**Validated**

The Smart Campus Event Hub MVP achieved an **86% task success rate**, a **4.7/5 usefulness score**, and a **4.7/5 interest level**, demonstrating that the prototype provides meaningful value to students, organizers, and administrators.

The validation confirmed that the core product direction is correct. Future improvements should focus on usability refinement, especially search, filtering, registration, and dashboard improvements rather than changing the overall MVP concept.
