# Business Model Canvas

## Project Title

**Smart Campus Event Hub – Campus Event Discovery and Registration Platform**

---

# 1. Customer Segments

The main users of Smart Campus Event Hub are:

- **First-year undergraduate students** who need an easier way to discover campus events, workshops, competitions, and university activities.
- **International and exchange students** who are unfamiliar with university communication channels and need centralized event information.
- **Student clubs and organizations** that need an effective platform to publish and promote their events.
- **Campus administrators and university departments** responsible for managing and promoting student activities.

---

# 2. Value Proposition

Smart Campus Event Hub provides a centralized platform where students can easily discover, register for, and receive updates about university events from one location.

The platform helps students:

- Find upcoming campus events without searching across multiple platforms.
- Receive timely reminders about events and registration deadlines.
- Search and filter events based on category, date, organizer, and interests.
- Register for workshops, competitions, and campus activities easily.
- Increase participation and engagement in university life.
- Access reliable and organized event information.

The platform helps organizers:

- Publish campus events efficiently.
- Manage event information in one place.
- Reach more students and increase event participation.
- Reduce dependence on posters, messaging groups, and scattered announcements.

---

# 3. Channels

Users can access Smart Campus Event Hub through:

- University website integration
- Mobile-friendly web application
- GitHub Pages prototype demonstration
- QR codes during student orientation programs
- University social media channels
- Student clubs and organization announcements
- Campus digital communication platforms
- Word-of-mouth recommendations from students

---

# 4. Customer Relationships

Smart Campus Event Hub maintains relationships with users through:

- Student self-service event browsing
- Online event registration
- Search and filtering features
- Personalized event notifications
- Event reminders before scheduled activities
- Organizer event publishing tools
- Student feedback and participation records
- Continuous improvement based on user feedback

---

# 5. Revenue Streams / Sustainability Logic

Smart Campus Event Hub is designed as a university-supported digital platform rather than a commercial application.

Its sustainability comes from:

- Improving student engagement and participation in campus activities.
- Helping new students adapt to university life.
- Reducing information management workload for university staff.
- Supporting digital transformation initiatives within the university.
- Providing an organized communication channel between students and campus organizations.

Future opportunities may include:

- University funding and support.
- Partnerships with student organizations.
- Sponsored campus events.
- Premium promotion services for approved university activities.

---

# 6. Key Resources

The resources required include:

- HTML, CSS, and JavaScript web application.
- GitHub repository and GitHub Pages hosting.
- Campus event information and announcements.
- Student and organizer user feedback.
- Event database system (future development).
- Development team members.
- Documentation, testing, and validation evidence.
- University collaboration and support.

---

# 7. Key Activities

The main activities include:

- Developing and maintaining the Smart Campus Event Hub platform.
- Collecting and organizing campus event information.
- Creating event listing and registration features.
- Developing search and filtering functions.
- Managing event notifications and reminders.
- Supporting student registration processes.
- Testing usability and improving the user experience.
- Updating event information regularly.

---

# 8. Key Partners

Potential partners include:

- **Rangsit University Administration**
- **Student Affairs Office**
- **University IT Department**
- **Student clubs and organizations**
- **Orientation Committee**
- **International Student Office**
- **Faculty departments**
- **Event organizers and campus communities**

---

# 9. Cost Structure

Expected costs include:

- Prototype development time.
- HTML, CSS, and JavaScript development resources.
- GitHub hosting (free for MVP).
- Testing and debugging activities.
- Documentation preparation.
- Event data management.
- Future database and cloud hosting services.
- System maintenance and updates.

---

# Final Note

This Business Model Canvas supports the Smart Campus Event Hub MVP by aligning customer needs, value proposition, and operational activities with the problem identified during customer discovery.

The platform addresses the challenge of scattered campus event information by providing a centralized solution where students can discover activities, register for events, receive reminders, and stay connected with university opportunities. At the same time, student clubs and administrators gain an efficient way to publish and manage campus events, creating a more connected and engaged university community.
