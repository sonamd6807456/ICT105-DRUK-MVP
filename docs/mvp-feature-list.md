# Lab 04 - MVP Feature List

## MVP Decision Rule
The MVP should include the smallest set of features that can demonstrate the core problem-solution fit by helping students discover campus events from a centralized platform.

## MoSCoW Prioritization

| Feature ID | Feature Name | Problem Solved | Related User Story | Priority (Must/Should/Could/Won't) | Technical Complexity (1-5) | User Value (1-5) | Evidence Strength (1-5) | Include in Final Prototype? |
|---|---|---|---|---|---:|---:|---:|---|
| F01 |Event Listing Page |Students cannot easily find campus events | US-01 | Must | 2 |  5 | 5 | Yes |
| F02 |Event Details Page |Event information is incomplete or difficult to access | US-02 | Must | 2 | 5 | 5 | yes |
| F03 |Search Events |Students spend time searching for events | US-03 | Must | 3 | 4 | 4 | yes |
| F04 |Event Categories Filter |Students want to find relevant events quickly | US-04 | Should | 3 | 4 | 4 | yes |
| F05 |Event Submission Form |Organizers need a way to add events | US-05 | Should | 3 | 4 | 4 | yes |
| F06 |Event Dashboard Summary |Users want a quick overview of upcoming events | US-06 | Could | 4 | 3 | 3 | optional |
| F07 |Event Reminder Notifications |Students miss events due to late reminders | US-07 | Won't | 5 | 4 | 3 | no |
| F08 |Personalized Recommendations |Students want tailored event suggestions | US-08 | Won't | 5 | 3 | 2 | no |


## Must-Have Features

* Event Listing Page
* Event Details Page
* Search Events

## Should-Have Features

* Event Categories Filter
* Event Submission Form

## Could-Have 
* Dashboard SummaryY

## Future Features
* Event Reminder Notifications
* Personalized Event Recommendations
* AI-based Event Suggestions
* User Accounts and Authentication
* Calendar Synchronization
* Mobile Application Version

## Not in MVP

The following features are intentionally excluded from the MVP because they require more development time and are not essential for validating the core problem-solution fit:

* Event Reminder Notifications
* Personalized Event Recommendations
* AI-based Event Suggestions
* User Accounts and Authentication
* Calendar Synchronization
* Mobile Application Version

The MVP will focus on helping students easily discover and access campus event information from a single platform.
